【第1章】 基本設定とUIアイコンChapter 1: Basic Setup & UI

⌨️

キー割り当てのルール (Your Keys / Original Keys) 🆓

Key Binding Rules 🆓

対象サイトの「✏️ 編集」を押し、あなたが実際に使いたい送信キー・改行キーを設定します。これだけで基本的なショートカットの統一が完了します。

Edit the target site and set your preferred send/newline keys to unify your shortcuts.

👉 詳しい設定項目を見る👉 View Detailed Setup
  • ① 送信キー (自分): あなたが実際に使いたい理想の送信キー（例: Shift+Enter）。
  • ② 改行キー (自分): あなたが改行に使いたいキー（例: Enterのみ）。
  • ③ 送信 / ④ 改行キー (元々): そのサイトが元々標準で採用しているキー。当アプリがこの本来のキー信号を偽装してサイトに送るために、必ず正しく設定しておく必要があります。
  • ① Send / ② Newline (You): Your preferred keys.
  • ③ Send / ④ Newline (Original): The site's default keys. The app needs these to fake the signal.
🎯

自動取得 と スマートターゲット 🆓

Auto Select & Smart Target 🆓

プログラムが送信ボタンの場所（CSSセレクタ）を特定するための強力な補助ツール群です。ご自身でコードを書く必要はありません。

Tools to automatically detect the CSS selector of the send button without coding.

👉 各アイコンの機能を見る👉 View Icon Functions
  • 🎯 自動取得 (Auto Select): このボタンを押したあと、実際のWebページの「送信ボタン」をクリックするだけで、アプリが自動でCSSセレクタ（ボタンの住所）を解析して入力してくれます。
  • ⚡ スマートターゲット: 送信ボタンのIDが毎回ランダムに変わってしまう厄介なサイト向けの機能です。「現在フォーカスしているテキストエリアの周辺」から、送信ボタンらしい要素をプログラムが推測して見つけます。
📋

マルチCSSセレクタ (複数入力欄の対応) 🆓

Multi-CSS Selectors 🆓

ChatGPTやSlackのスレッドのように、1つの画面に「送信欄」が複数存在するサイトで使用します。動的検知をONにすることで、現在使っている入力欄のボタンを正確に狙い撃ちします。

For sites with multiple inputs on one screen (like Slack threads or ChatGPT).

👉 設定方法を見る👉 View Setup Steps
「複数セレクタの動的検知を有効にする」をONにし、最大5つまで登録枠を増やします。対象のタブで「🎯 自動取得」を押すと、青いツールチップがマウスに追従します。それぞれの入力欄の送信ボタンを順番にクリックして登録してください。

送信方式 (全15種類) 🆓

Send Methods (15 Types) 🆓

Webアプリの仕様に合わせて「プログラムでどうやって送信ボタンを押すか」のロジックを選びます。通常は「CSSクリック」で動作しますが、送信できない場合はここを変更します。

Choose the backend logic matching the web app. Check "Show Advanced Methods" to enable all 15 powerful methods to bypass any block.

👉 15種類の詳細一覧を開く👉 Show all 15 methods
CSSクリック 基本画面上の送信ボタンをプログラム的にクリック (click()) します。大半のサイトはこれで動きます。
信号偽装 基本サイト本来の送信キー信号を同期的にブラウザに送ります。
強力CSSクリック 高度人間のマウス操作（mousedown → mouseup）を精密に偽装します。
強力信号偽装 高度KeyUpイベントを遅延送信し、監視の厳しいサイトに対応します。

完全非同期 信号 高度タイピングの速度差を完全再現する非同期シミュレーションです。
プロパティ完全一致 高度キーボードイベントの内部プロパティを本物と完全に一致させます。
Form直接送信 高度ボタンではなく、親のFormタグに対して直接Submit命令を出します。
InputEvent偽装 高度キーボードイベントを使わず、Inputイベントのみを送信します。
親要素バブリング 高度対象要素の親DIVへイベントを送信し、バブリング（伝播）を利用します。

メイン世界からの発火 究極拡張機能の空間を抜け出し、サイト本体のメモリ空間にスクリプトを注入します。

【第4章】 誤爆防止機能Chapter 4: Mistake Protection

⏳

送信カウントダウン機能 👑

Send Countdown Feature 👑

誤爆を防ぐため、送信ボタンを押した直後に指定した秒数（例: 3秒）のカウントダウンを開始します。宛先間違いに気づいたら「キャンセル」で止められます。

Start a countdown before sending. Click cancel if you notice a mistake.

👉 便利なスキップ設定について👉 Skip Settings
設定画面で「もう一度はいを押せばすぐに送信できる」をONにすると、カウントダウンの待機中であっても、もう一度「はい」のボタンを押すことで待たずに即時送信が可能になります。テンポを落とさずに安全を確保できます。
🚫

NGワード検知 🆓

NG Words 🆓

登録した言葉（社外秘、機密など）が含まれている場合、送信時に検知して警告を出したり、送信を完全にブロックしたりできます。

Blocks or warns when registered forbidden words are detected in the message.

👉 動作の選択について👉 Action Options
検知時のアクションは「強制ブロック（一切送信させない）」か「警告して確認（赤文字のダイアログを出し、もう一度『はい』を押せば送信できる）」の2種類から選べます。
🛑

時間帯送信ブロック (タイムフィルター) 👑

Time-based Send Block 👑

指定した時間帯（例：22:00〜08:00など）の送信操作を完全にブロックします。働き方改革や、深夜のテンションでの誤送信を防止するのに役立ちます。

Completely blocks sending during specified hours to prevent accidental off-hours messaging.

【第7章】 仮想キーボード連携Chapter 7: Virtual Keyboard Host

💻

[Windows専用] 仮想キーボード連携 🆓

[Win Only] Virtual Keyboard Host 🆓

JavaScriptによる操作を完全に弾く超・強固なサイトにおいて、Windows OSレベルで直接ハードウェアキーボードの信号（物理的なエンターキー）を叩くための究極機能です。

The ultimate feature to physically tap hardware keys via Windows OS, bypassing strict JS blockers.

👉 安全な設計とセットアップ手順👉 Setup & Safety

■ 安全設計について

この連携アプリはキーボードの入力を盗み見る「キーロガー」ではありません。送信の瞬間だけ命令を受け取る安全な設計であり、ブラウザ終了と共に通信が即座に切断されます。

■ セットアップ手順

  1. 設定画面にて「💻 究極突破：Windows仮想キーボードを使用して物理送信する」にチェック。
  2. 専用アプリ(ZIP)をダウンロードし、exeを展開。
    ※EXEを直接ダブルクリックしてエラーが出るのは正常です（Chromeからの通信を待機するため）。
  3. exeのフルパスを入力し「⬇️ 登録用(.bat)生成」をクリック。
  4. ダウンロードされたbatファイルを「右クリックして、管理者として実行」。
  5. F5キーで再読み込みし、設定画面に「🟢 接続済 (Connected)」が出れば完了です。

【第10章】 よくある質問 (Q&A)Chapter 10: Frequently Asked Questions 🆓

Q. 設定を保存したのに、チャット画面で動きません。Q. I saved but it doesn't work on the chat screen.
設定を保存した後は、対象のチャット画面（タブ）を一度「リロード（F5キーまたは更新ボタン）」してください。
それでも動かない場合は、現在の拡張機能のモード（Private / Work）が、設定したプロファイルのモードと一致しているか確認してください。
Always reload (press F5) the target chat tab after saving. Also, ensure your Global Mode matches the profile mode.
Q. 送信ボタンが反応しない、改行がおかしくなります。Q. The send button doesn't react, or newlines break.
サイトによって内部の作りが異なります。「✏️ 編集」画面から以下をお試しください。
・送信されない時： 第2章「送信方式」を『CSSクリック』や『信号偽装』等に変える。
・改行文字が消える時（Teams等）： 第3章「システム文字入力方式」を『仮想ペースト』に変える。
・どうしてもダメな時： 「高度な方式を表示」にチェックを入れ、『[最強] React Fiber仮想DOM走破』などの特殊方式を試す。
Try changing the "Send Method" to "CSS Click" or "Signal". For newline bugs (like in Teams), change "Text Input Method" to "Virtual Paste".
Q. 「🎯 自動取得」を使ってもボタンが登録されません。Q. "Auto Select" doesn't capture the button correctly.
一部の特殊な動的サイト（SPAなど）では自動取得が効かない場合があります。その場合は、ブラウザの「検証（F12）」ツールを使ってボタンのCSSセレクタ（class名やidなど）をご自身で調べ、手動で入力していただく必要があります。
On complex sites (like SPAs), auto-select might fail. You must use browser Inspect (F12) to find the CSS selector manually.
Q. 翻訳ウィジェット(🌐)を押しても反応しません。Q. The translation widget doesn't work.
翻訳機能は外部の翻訳API通信を使用しています。入力欄に文字が入っていないか、長文すぎる場合、または短時間に連続して押しすぎてAPIの制限に引っかかっている可能性があります。少し時間を置いてお試しください。
The translation uses an external API. It may fail if the text is empty, too long, or if you hit the API rate limit by clicking too fast.
Q. 復元ウィジェット(🕒)の文字数が足りない気がします。Q. The restore widget seems to miss some text.
ブラウザの動作が重くなるのを完全に防ぐため、バックアップされるテキストは「最新の 50,000文字（5万文字）」までに安全制限されています。5万文字を超える超長文の場合は、末尾（または古い部分）がカットされている可能性があります。
To prevent browser lag, the auto-backup safely limits the text to 50,000 characters. Extra long texts may be truncated.
Q. 別のパソコンでも同じ設定を使えますか？Q. Can I use the same settings on a different computer?
はい。ご自宅のPCと会社のPCなど、同じGoogleアカウントでChromeにログインして「同期」をオンにしていれば、Unlimited版のライセンス認証状態は自動的に引き継がれます。
※細かいサイト設定（プロファイル）を移す場合は、設定画面下部の「エクスポート」でJSONファイルを保存し、別のPCで「インポート」してください。
Yes. As long as you log into Chrome with the same Google Account and Sync is enabled, your Unlimited license status will sync.
Q. 会社のPCで「仮想キーボード(.bat)」が実行できません。Q. Virtual Keyboard Host gives an error on company PC.
第9章の「仮想キーボード連携（.batの実行）」機能はWindowsのレジストリを書き換えるため、会社の厳しいセキュリティ設定やウイルス対策ソフトでは弾かれることがあります。
その場合は連携機能のご利用を諦め、「送信方式」から『仮想ペースト』や『React Fiber方式』などのソフトウェア完結の方式をお試しください。
Modifying the Windows Registry is often blocked by corporate security. Please use alternative JS-based methods like "React Fiber" instead.
Q. Macでもすべての機能を使えますか？Q. Does it work fully on Mac?
Chrome拡張機能としての基本機能（ショートカット統一、ウィジェット、カウントダウン等）はMacでも完全にご利用いただけます。
ただし、[Windows上級者向け]の「仮想キーボード連携（Native Host）」機能のみ、Windowsのシステム（.exe / .bat）に依存しているためMacではご利用いただけません。
Yes, all core extension features work perfectly on Mac. However, the [Advanced] "Virtual Keyboard Host" is strictly Windows-only.
Q. 拡張機能のアイコンがツールバーに見当たりません。Q. The extension icon is missing from the toolbar.
Chromeの右上にある「パズルのピース」のアイコンをクリックし、リストの中から「Custom Enter Pro」の横にある「ピン留め（📌）」アイコンをクリックして青くすると、常にツールバーに表示されるようになります。
Click the "Puzzle Piece" icon in the top right of Chrome, find "Custom Enter Pro", and click the "Pin" icon next to it.
